# 마이링크 (MyLink) 기능 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 개발자, 크리에이터, 보안 전문가 등의 타겟 유저가 운영하는 채널 및 포트폴리오 링크를 하나의 프로필 페이지로 모아 직관적으로 공유할 수 있는 심플한 멀티 링크 서비스.
- **대상 사용자**: 복잡한 꾸미기 기능을 배제하고, `shadcn/ui` 기반의 깔끔하고 전문적인 인터페이스를 선호하는 집단.

## 2. 세부 요구사항 (MVP Core)

### 2.1. 사용자 인증 (Auth)
- **Firebase Google Login 단일화**: 이메일 등 별도의 계정 생성 과정 없이, Firebase Authentication을 활용한 구글(Google) 계정 소셜 기반 로그인만 지원.

### 2.2. 프로필 관리 및 데이터베이스 구조
- **프로필 정보 구성**: 
  - **표시 이름 (displayName)**: 서비스의 URL 슬러그(Slug)이자 화면 전면에 표시되는 고유 타이틀. 소셜 로그인 시 **유저의 구글 지메일(Gmail) 아이디 앞부분을 추출하여 기본값으로 자동 설정**됨. 
  - **사용자 이름 (username)**: 실제 사용자의 이름(실명 등 본명)을 나타내는 보조 텍스트 정보 필드.
  - **소개글 (Bio)**: 짧은 자기소개 텍스트.
- **단일 URL 라우팅**: 별도의 복잡한 영문 핸들 생성 프로세스 없이, 식별자인 `displayName` 값 자체를 퍼블릭 접근 경로(Path)로 즉시 활성화. (예: `domain.com/displayName`)
- **프로필 정보수정 (인라인 편집)**: `displayName`, `username`, `Bio` 텍스트를 변경할 때 다른 페이지로 이동하거나 팝업을 띄우지 않고, 화면의 텍스트 자체를 마우스로 클릭하여 폼으로 자연스럽게 전환 시키는 **인라인 편집(Inline Editing)** 방식 단독 적용. 
- **데이터베이스 모델링 (Firebase Firestore)**:
  - 사용자 로그인 및 가입 시 할당되는 메인 도큐먼트(Document) 구조 내부에 `displayName`과 `username` 필드를 모두 필수로 포함하여 저장.
  - 각 링크 정보들은 하나의 도큐먼트 안에서 배열 구조로 섞이는 대신, 물리적인 분리가 이루어지도록 해당 유저 도큐먼트 하위의 **서브 컬렉션(Subcollection)**으로 종속시켜 데이터 독립성과 보안성 관리.

### 2.3. 링크 관리 기능 (CRUD)
- **명시적 링크 등록**: 생성할 페이지를 특정하는 `URL` 문자열과 화면상 버튼 안에 표기될 `표시 제목(Title)` 텍스트의 입력 및 리스트 삭제 처리.
- **인라인 편집 지원**: 등록된 각각의 링크 텍스트 및 연결 URL 값 역시, 사용자가 텍스트를 직접 클릭해 그 자리에서 즉각 수정하는 인라인 편집(Inline Editing)을 허용하여 관리 편의 극대화.
- **구글 파비콘 API (Favicon Parsing)**: 유저가 새 URL을 폼에 삽입 등록할 시 **구글 파비콘 API(Google Favicon API)**를 백그라운드 호출. 대상 외부 사이트의 깔끔한 로고 썸네일을 반환받아 서비스 내 해당 링크의 대표 아이콘(Icon)으로 영구 적용 및 파싱.

### 2.4. 퍼블릭 공유 페이지 (Public View)
- **접속 뷰**: `domain.com/displayName` 형태로 외부 사용자 대상 노출 및 렌더링.
- **디자인 시스템**: 테마 변경 기능을 철저히 배제. 초기 구성 시 렌더링 된 `shadcn/ui` 베이스 단일 디자인 컴포넌트 생태계로 프론트엔드 전반의 시각 요소를 완벽히 통일.

## 3. 기능 제외 대상 스펙 및 향후 릴리즈 계획

### 3.1. MVP 명시적 제외 스펙 (Not In Scope)
- 프로필 이미지 파일 업로드 기능 (텍스트 및 링크 파비콘으로 대체)
- 구동 시 별도 인증/소비를 위한 복합 핸들(ID) 및 고유 닉네임 추가 시스템 채택 
- 테마 색상 변경 등 전면적인 디자인 자유 커스터마이징 기능
- 사용자 단위별 방문자 누적 확인 등 광범위 통계 인터페이스
- 관리자 패널 창 내부의 모바일 뷰 미리보기(Preview) 위젯 엔진
- 등록된 개별 링크에 대한 강제 숨김/표시 (On/Off 토글 버튼)
- 마우스 드래그 앤 드롭 방식을 이용한 블록 순서 및 무결성 제어 기능
- 특정 기술분야만을 위주로 표적 지향하여 구현하는 커스텀 노출 위젯

### 3.2. 향후 추가 릴리즈 스펙
- **단일 링크 클릭 수 통계 조회**: 전역 방문 통계는 배제되었으나, 이후 개별 링크들이 각각 몇 번 눌렸는지 누적하여 기록하는 단건 단위의 '클릭 조회 수 집계'는 향후 패치나 시즌2 뷰를 통하여 제한적으로 부가해 나갈 계획 존재.
