# 마이링크 (MyLink) 사용자 시나리오 (User Scenario)

## 1. 신규 가입 및 프로필 초기화 시나리오
1. **사이트 접속**: 사용자는 브라우저에 접속하여 첫 화면 중앙의 'Google로 로그인' 리드 버튼을 클릭한다.
2. **소셜 인증 기반 자동 로직**: Firebase Authentication이 띄워주는 Google 로그인 창에서 본인의 구글 계정을 선택해 로그인을 완전 완료한다.
3. **핵심 온보딩 및 자동 추출**: 서비스 로그인 즉시, 시스템은 해당 구글 계정의 이메일(Gmail) 아이디에서 `@` 앞부분(Prefix)을 온전하게 추출하여 **`displayName(표시 이름 겸 URL 슬러그)`** 기본값으로 고정 세팅 및 렌더링한다.
   - 예: `minjong@gmail.com` 으로 로그인 -> 초기 displayName: `minjong` 추출 처리됨. (향후 퍼블릭 접속 주소: `domain.com/minjong` 사용)
4. **대시보드 진입**: 사용자는 URL 슬러그의 뼈대인 `displayName`, 자동 연동 입력된 구글 계정의 실명인 `username`, 그리고 비어 있는 `Bio(자기소개)` 골격이 구조적으로 화면에 그려진 메인 관리자 대시보드 화면으로 즉각 안착한다.

## 2. 프로필 정보 수정 (인라인 편집) 시나리오
1. **호버 힌트 제공**: 대시보드 화면 상단에 렌더링된 본인의 `displayName`, `username`, 관리 및 수정 제어용 `Bio` 문구 텍스트 영역에 사용자가 마우스 커서를 가볍게 올린다.
2. **시각적 힌트 투사**: 텍스트 영역 테두리가 옅어지며 입력 가이드 힌트가 나타나 해당 공간이 통째로 텍스트 폼 구실을 하는 수정 가능한 필드임을 직관적으로 확인한다.
3. **인라인 변경 및 안정성 검증**: 텍스트를 클릭하면 즉시 편한 느낌의 입력(Input) 폼으로 변환된다. `displayName`을 다른 영문 문자열로 수정하거나, 빈 `Bio` 뼈대를 채운 뒤, `Enter` 키를 치거나 바깥쪽 빈 여백 공간을 마우스로 누르면 클릭 한 번 없이 화면 깜박임도 전면 차단되며, Firebase Firestore DB 도큐먼트에 실시간 저장되어 적용 및 라우팅 경로가 즉시 변경 완료된다.

## 3. 링크 등록 및 업데이트 시나리오
1. **신규 행위의 생성**: 대시보드 내의 `+ 링크 추가` 커다란 유도 버튼을 클릭하여 완전 빈 형태의 등록 폼 행(Row) 하나를 즉각 생성한다.
2. **직관적인 정보 입력**: 점프 시킬 대상 웹 사이트의 물리적 `웹 주소(URL)`와, 외부 방문자에게 보일 버튼의 텍스트 `표시 제목(Title)` 글을 기입한다.
3. **완전 자동화된 파비콘(Favicon) 연동**: 사용자가 URL 입력을 완료하는 즉시 백그라운드 환경에서 **구글 파비콘 API(Google Favicon API)**가 내부적으로 백그라운드 호출된다. 등록 대상 사이트의 썸네일/로고 이미지가 덜컥 추출되어 링크 버튼의 좌측 아이콘으로 깔끔하게 자동 렌더링 반영된다.
4. **간소 링크 변경/삭제 프로세스**: 이미 등록된 완성 링크 역시 '인라인 편집' 방식으로 제목 텍스트나 URL 텍스트 덩어리를 즉시 클릭해 그 자리에서 딜레이 없이 변경 가능하다. 가치가 없어진 링크는 행 우측 끝 외곽에 표시된 휴지통(Delete) 버튼을 눌러 직관적으로 Firestore 서브 컬렉션(Subcollection) 내역 채 영구 파기시킨다.

## 4. 퍼블릭 공유 페이지 방문 시나리오 (Public View Test)
1. **외부 접속**: 마이링크 회원이 SNS 이력서 등에 뿌린 고유 공유 링크(예: `domain.com/minjong` 등 displayName 기반 라우팅) 문자열을 타인이 클릭하여 모바일 웹 페이지에 접속한다.
2. **명확하게 식별되는 프로필 정보**: 조잡한 사진이나 커스텀이 없는 깔끔하게 비워진 텍스트 집중형 레이아웃 위에, 매우 굵고 큰 폰트를 두른 `displayName`이 가장 거대하고 세련되게 시선을 훔치며, 해당 이름 바로 하단부에 실제 프로필 오너의 `username`과 부드러운 `Bio` 텍스트 정보가 정제된 타이포그래피 형태로 방문자를 부드럽게 반기고 설명한다.
3. **아름다운 구도 배열 뷰 및 전환 액션**: `shadcn/ui`의 고도화된 미니멀 프레임 버튼 및 그 좌우측의 여백감, 그리고 버튼의 좌측 가장자리에 자동 배치/파싱되어 나타난 구글 API 파비콘 아이콘들을 쭉 훑어보고 흥미에 부합하는 링크 버튼을 방문자가 한 번 눌러, 타겟 목표 사이트로 아무런 장애물 없이 원스텝 랜딩한다.
